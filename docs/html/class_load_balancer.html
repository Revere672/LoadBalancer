Detailed Description

Distributes incoming network requests across a dynamic pool of web servers.

A LoadBalancer owns a request queue and a vector of WebServer objects. It is identified by a single character name (e.g., 'P' or 'S'). On each clock cycle it:

  1. Accepts newly generated requests and pushes them onto its queue.
  2. Dispatches queued requests to idle servers (first-come, first-served).
  3. Calls update() on every server to advance their processing timers.
  4. Periodically evaluates the queue length relative to the server count and allocates or deallocates servers to maintain balance.

void LoadBalancer::runCycle ( std::vector< Request > * newRequests,
std::ofstream & logFile )

Executes a single clock cycle of the load balancer.

Runs one full clock cycle of the load balancer simulation.

Enqueues all requests in newRequests, dispatches work to available servers, updates server states, and (every cooldownTime cycles) auto-scales the server pool. Progress is logged to both stdout and logFile.

Parameters
newRequestsPointer to a vector of requests generated this cycle (vector is consumed during the call).
logFileReference to an open output stream for persistent logging.

The cycle proceeds in the following order:

  1. All requests in newRequests are pushed onto the internal queue.
  2. Each idle server is given the next request from the front of the queue.
  3. Every server's update() method is called to decrement processing timers.
  4. If the current clock time is a multiple of cooldownTime, the queue depth is compared against the scaled thresholds to decide whether to allocate or deallocate a server.
  5. The clock counter is incremented.
Parameters
newRequestsPointer to a vector of requests produced this cycle; elements are popped from the back and pushed to the queue.
logFileOpen output file stream used for persistent event logging.
